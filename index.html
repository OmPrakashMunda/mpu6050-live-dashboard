// HOW IT WORKS

MPU6050 DASHBOARD — DOCUMENTATION

This dashboard reads 6-axis IMU data from an MPU6050 sensor connected to an ESP8266. The ESP streams JSON over WebSocket at ~20Hz. The browser receives it, computes orientation via a complementary filter, renders a live 3D model using Three.js, and displays motion intensity on a canvas-drawn gauge.

01 / MPU6050 — Sensor Basics

The MPU6050 is a 6-axis IMU with a 3-axis accelerometer and 3-axis gyroscope on one chip. It communicates over I²C. The AD0 pin selects the I²C address: 

AD0 → GND  :  I²C address = 0x68  (default)
AD0 → VCC  :  I²C address = 0x69  (use for two sensors on same bus)

Raw 16-bit integers are scaled to real units:

Accel (g) = raw_value / 16384.0   [±2g range]
Gyro (°/s) = raw_value / 131.0   [±250°/s range]

02 / ESP8266 — Data Streaming

The ESP8266 reads MPU6050 via I²C (SDA=D2, SCL=D1) and broadcasts a JSON packet over WebSocket on port 81 every 50ms. No server needed — browser connects directly to the ESP on the same WiFi network. 

{ "ax": 0.012, "ay": -0.004, "az": 0.998,
  "gx": 0.31,  "gy": -0.12,  "gz": 0.08  }

03 / Complementary Filter — Orientation

Gyroscope is fast but drifts. Accelerometer is drift-free but noisy. A complementary filter blends both — 96% gyro for short-term, 4% accel for long-term correction.

angle = α × (angle + gyro × dt) + (1 − α) × accel_angle
accelRoll  = atan2(ay, az)
accelPitch = atan2(-ax, sqrt(ay² + az²))

roll  = 0.96 × (roll  + gx × dt × π/180) + 0.04 × accelRoll
pitch = 0.96 × (pitch + gy × dt × π/180) + 0.04 × accelPitch
yaw  += gz × dt × π/180   // gyro only — no accel reference for yaw

⚠ Yaw drifts over time — there's no absolute reference without a magnetometer. Use MPU9250 or HMC5883L for drift-free yaw.

04 / Motion Gauge — The Math

Shows net dynamic acceleration — motion force excluding static gravity.

magnitude = √(ax² + ay² + az²)   [in g]
net_g = | magnitude − 1.0 |   [subtract 1g gravity]
net_ms² = net_g × 9.81   [convert to m/s²]

Exponential moving average smooths the needle:

smoothG = 0.8 × smoothG_prev + 0.2 × net_g

05 / 3D Rendering — Three.js

The board is a BoxGeometry mesh rendered via WebGLRenderer. Each frame its Euler rotation is set directly from the complementary filter output: 

board.rotation.set(pitch, yaw, roll)  // Three.js XYZ Euler order
